# Deriving Arbitraries

Create new arbitraries by transforming, filtering, or chaining existing ones.

## `.map(mapper, unmapper?)`

Transform every generated value through a `mapper` function. This is the most common and safest way to derive a new arbitrary.

**Signatures:**

- `.map(mapper)` — transform values, no shrinking of user-defined values
- `.map(mapper, unmapper)` — transform values with shrinking support for user-defined values

**with:**

- `mapper` — `(t: T) => U` — transform a generated value into the target type
- `unmapper` — `(possiblyU: unknown) => T` — reverse the mapping; must throw if the value is incompatible. Only called when shrinking user-defined values (since 3.0.0)

**Examples:**

```ts
// Square numbers
fc.nat(1024).map((n) => n * n);

// Change type from number to string
fc.nat().map((n) => String(n));

// Generate an ordered range [min, max]
fc.tuple(fc.integer(), fc.integer()).map((t) => (t[0] < t[1] ? [t[0], t[1]] : [t[1], t[0]]));

// RGB channels to hex color string
const rgbChannels = fc.record({ r: fc.nat(255), g: fc.nat(255), b: fc.nat(255) });
const hexColor = rgbChannels.map(
  (ch) => `#${(ch.r * 65536 + ch.g * 256 + ch.b).toString(16).padStart(6, '0')}`,
);
```

**When to provide an `unmapper`:**

The `unmapper` is only needed when you want fast-check to be able to shrink values that were not generated by this arbitrary (e.g. user-defined counterexamples). It must throw if the input is not a valid mapped value.

```ts
fc.nat().map(
  (n) => String(n),
  (s) => {
    if (typeof s !== 'string') throw new Error('not a string');
    return Number(s);
  },
);
```

Resources: [API reference](https://fast-check.dev/api-reference/classes/Arbitrary.html#map).
Available since 0.0.1.

## `.filter(predicate)`

Keep only values satisfying a predicate. All generated and shrunk values will pass the predicate.

> **Warning:** Internally, `.filter` uses an infinite retry loop. If the predicate rarely matches, generation can hang or become extremely slow. Always prefer built-in constraints or `.map` tricks when possible.

**Signatures:**

- `.filter(predicate)` — keep values where `predicate(value) === true`
- `.filter(refinement)` — narrow the type using a TypeScript type guard

**Examples:**

```ts
// Only even integers
fc.integer().filter((n) => n % 2 === 0);

// Only strings where the first character is less than the second
fc.string().filter((s) => s[0] < s[1]);
```

**Prefer alternatives over `.filter`:**

| Instead of | Use |
|---|---|
| `fc.integer().filter(n => n >= 1)` | `fc.integer({ min: 1 })` |
| `fc.integer().filter(n => n >= 0)` | `fc.nat()` |
| `fc.string().filter(s => s.length >= 2)` | `fc.string({ minLength: 2 })` |
| `fc.integer().filter(n => n % 2 === 0)` | `fc.nat().map(n => n * 2)` |

Resources: [API reference](https://fast-check.dev/api-reference/classes/Arbitrary.html#filter).
Available since 0.0.1.

## `.chain(chainer)`

Create a new arbitrary that depends on a previously generated value (flat-map). The `chainer` function receives a generated value and returns a new arbitrary.

> **Warning:** The shrinker of `.chain` constructs might not shrink as much as possible. This is a known limitation. Prefer `.map`, `fc.record`, or `fc.tuple` when the values are independent.

**Signatures:**

- `.chain(chainer)` — `chainer: (t: T) => Arbitrary<U>` — produce a new arbitrary from a generated value

**Examples:**

```ts
// Generate a valid range [min, value] where value >= min
fc.nat().chain((min) => fc.tuple(fc.constant(min), fc.integer({ min, max: 0xffffffff })));
```

**When to use `.chain`:**

Only use `.chain` when the second arbitrary genuinely depends on the first generated value. If values are independent, prefer composition:

```ts
// Independent values: use fc.record or fc.tuple, NOT .chain
fc.record({ name: fc.string(), age: fc.nat(120) });

// Dependent values: .chain is appropriate
// Generate a string and a valid index into that string
fc.string({ minLength: 1 }).chain((s) => fc.tuple(fc.constant(s), fc.nat(s.length - 1)));
```

Resources: [API reference](https://fast-check.dev/api-reference/classes/Arbitrary.html#chain).
Available since 1.2.0.
